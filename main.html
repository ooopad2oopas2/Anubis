FAQ: How Anubis fits with Etez / ElonaInterface only – no private keys, no signing, no blockchain calls.

Anubis is the human-facing lens on top of Etez / Elona. It retrieves institution definitions and time series from the Etez API, renders a minimal chart and aggregates, and lets you insert new institutions and snapshots.

All persistence, invariants, and any smart-contract plumbing live beneath the surface in your Etez / Elona stack. Anubis never touches wallets, signing, or on-chain connectivity.

Anubis talks to a single base URL, configured as http://localhost:8097 by default. From there, it expects a minimal, JSON-based API:

  • GET /institutions → list all controllers
  • GET /institutions/:id/series → time series for a controller
  • GET /institutions/:id/aggregates → derived metrics for a controller
  • POST /institutions → create a new controller
  • POST /institutions/:id/snapshots → append a new flow snapshot

If your Etez instance uses a different path layout, you can either adapt the UI or layer a compatibility shim in front of the API.

Operational notesIntended for a single operator console or desk view.

This page keeps no durable state beyond what Etez / Elona provides. If you reload, Anubis re-queries your API and rebuilds the view. If you need multi-operator audit trails or permissions, build them into your backend.

The activity log is session-scoped only by design. It helps you see how the current operator interacted with the system but is not a substitute for a real audit ledger.
